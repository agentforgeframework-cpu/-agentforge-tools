# CTS — Capture–Clean–Think–Share

## What This Is

CTS is a simple, practical workflow for turning messy, real-world input into clear, actionable understanding and shareable output.

It is designed for:
- incomplete notes  
- fast-paced meetings  
- transcripts and recordings  
- mixed input (text, images, audio)  

---

## The Workflow

Capture → Clean → Think → Share

### 1. Capture — NoteIntake

Gather information quickly without trying to organize it.

* Accept messy input
* Stay in intake mode
* Do not interpret or summarize

---

### 2. Clean — NoteCleanup

Turn messy input into structured, usable notes.

* Extract signal from noise
* Reconstruct meaning
* Identify key ideas and next steps

---

### 3. Think — DeepDive

Apply structured reasoning to understand the material.

* Analyze assumptions and risks
* Identify patterns and insights
* Produce actionable direction

---

### 4. Share — VisualBrief

Create a clean, professional summary for others.

* Visual summary (preferred)
* Slide or document fallback
* Clear, accurate, and concise

---

## How to Use

### Quick Start

1. Paste messy input
2. Run:

   * `NoteCleanup`
   * `DeepDive`
   * `VisualBrief`

### Full Workflow (recommended)

1. Activate `NoteIntake` and capture information
2. Deactivate intake mode
3. Run `NoteCleanup`
4. Run `DeepDive`
5. Run `VisualBrief`

## Files

* `NoteIntake.md` — Rapid information capture
* `NoteCleanup.md` — Structured note reconstruction
* `DeepDive.md` — Structured analysis and reasoning
* `VisualBrief.md` — Visual summary generation

---

## When to Use CTS

Use CTS when:

* input is messy or incomplete
* clarity is needed
* decisions or insights are required
* results must be shared with others

Do not use CTS for:

* simple factual lookups
* already clean and structured documents

---

## Practical Tips

* Do not pre-clean input before using the system
* Deactivate NoteIntake before analysis
* If output feels off, run again with a simple clarification
* If needed, use a fresh chat session for clean execution
* Let the tools do their job—avoid skipping steps
