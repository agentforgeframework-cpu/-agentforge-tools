# Decision Guide

This guide identifies where human judgment matters.

---

## Decision 1 — Is this worth capturing?

The operator decides whether the troubleshooting session produced reusable knowledge.

Good reasons to capture:
- issue may recur
- fix was non-obvious
- environment mattered
- error message was confusing
- future operator may need context

---

## Decision 2 — What actually fixed the issue?

The operator distinguishes between:
- actions that helped
- actions that were unrelated
- guesses that did not work
- the final confirmed fix

This prevents false recovery guidance.

---

## Decision 3 — How much detail is enough?

The operator should include enough detail for recovery, but not a full diary.

Minimum useful detail:
- symptom
- environment
- key evidence
- fix
- warning or gap

---

## Decision 4 — Where should the note live?

The operator chooses a location where future humans can find it.

The note should not disappear into private chat history or personal memory.
