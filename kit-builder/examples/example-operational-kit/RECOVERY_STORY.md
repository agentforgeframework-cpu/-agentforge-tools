# Recovery Story

You open the troubleshooting notes folder and cannot find a note for the issue you remember solving.

First, check whether the note was saved under a different symptom name.

If that fails, search for:
- the error message
- the tool name
- the command that failed
- the date of the session
- related project folders

If no note exists, create a new note from whatever evidence remains.

Mark it clearly as reconstructed:

```text
Status: Reconstructed from partial memory/evidence
```

Then document:
- what is known
- what is uncertain
- what should be verified next time

The goal is not to fake certainty.

The goal is to preserve the best available recovery path.
