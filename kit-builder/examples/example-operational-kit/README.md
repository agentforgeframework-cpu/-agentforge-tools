# example-operational-kit

Purpose:
Demonstrate the minimum survivable workflow for turning an operational idea into an AgentForge kit.

This example is intentionally small.

Operational idea:
A human wants a simple way to capture, review, and preserve quick troubleshooting notes from a work session so another person can understand what happened later.

This is not a production tool.
It is a teaching example for kit-builder.

---

# Minimum Survivable Artifacts

This example contains:

- STORY.md
- DECISION_GUIDE.md
- CONSTRAINTS.md
- STEPS.md
- RECOVERY_STORY.md
- KNOWN_GAPS.md

These match the minimum survivable workflow defined in:

```text
kit-builder/core/MINIMUM_SURVIVABLE_WORKFLOW.md
```

---

# How to Use This Example

Read the files in this order:

1. STORY.md
2. DECISION_GUIDE.md
3. CONSTRAINTS.md
4. STEPS.md
5. RECOVERY_STORY.md
6. KNOWN_GAPS.md

The point is not the example itself.

The point is to show how a small operational idea becomes a survivable kit.
